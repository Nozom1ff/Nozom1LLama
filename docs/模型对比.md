# Qwen与LLaMA模型工程实现对比分析

本文档基于 KuiperLLama 项目，详细分析 Qwen 系列模型与 LLaMA 系列模型在工程实现上的差异。

## 2. 模型架构对比

### 2.1 整体架构（相同）

两个模型系列都采用 **Decoder-only Transformer** 架构：

```
┌─────────────────────────────────────────────┐
│              Token Embedding                │
├─────────────────────────────────────────────┤
│  ┌───────────────────────────────────────┐  │
│  │         Attention Block               │  │
│  │  RMSNorm -> QKV Projection            │  │
│  │  -> RoPE -> Multi-Head Attention      │  │
│  │  -> Output Projection -> Residual     │  │
│  └───────────────────────────────────────┘  │
├─────────────────────────────────────────────┤
│  ┌───────────────────────────────────────┐  │
│  │          FFN Block                    │  │
│  │  RMSNorm -> W1/W3 -> SwiGLU           │  │
│  │  -> W2 -> Residual                    │  │
│  └───────────────────────────────────────┘  │
├─────────────────────────────────────────────┤
│         Final RMSNorm -> LM Head            │
└─────────────────────────────────────────────┘
```

### 2.2 层结构定义（相同）

**LLaMA 层结构** (`kuiper/include/model/llama3.h`):
```cpp

  std::shared_ptr<op::Layer> add_layer_;         // 残差加法
  std::shared_ptr<op::Layer> rope_layer_;        // RoPE 位置编码

  std::shared_ptr<op::Layer> mha_layer_;         // 多头注意力

  std::vector<std::shared_ptr<op::Layer>> wq_layers_;    // Q 投影层
  std::vector<std::shared_ptr<op::Layer>> wk_layers_;    // K 投影层
  std::vector<std::shared_ptr<op::Layer>> wv_layers_;    // V 投影层
  std::vector<std::shared_ptr<op::Layer>> wo_layers_;    // 输出投影层

  std::vector<std::shared_ptr<op::Layer>> w1_layers_;    // FFN 第一层
```

## 3. 关键差异分析

### 3.1 Attention Bias（核心差异）

**这是 Qwen 与 LLaMA 最重要的工程差异。**

#### LLaMA 实现（无 Bias）
```cpp
// kuiper/source/model/llama3.cpp
auto wq = std::make_shared<op::MatmulLayer>(device_type_, dim, dim);
wq->set_weight(0, {dim, dim}, this->raw_model_data_->weight(pos), cpu_device_type);
// 注意：没有 set_bias 调用
```

#### Qwen2 实现（有 Bias）
```cpp
// kuiper/source/model/qwen2.cpp
auto wq = std::make_shared<op::MatmulLayer>(device_type_, dim, dim, false, true);
wq->set_weight(0, {dim, dim}, this->raw_model_data_->weight(pos), cpu_device_type);
pos += dim * dim;
wq->set_bias(0, dim, this->raw_model_data_->weight(pos), cpu_device_type);  // Qwen 有 bias
pos += dim;
```

**差异总结：**
| 参数 | LLaMA | Qwen |
|------|-------|------|
| Q 投影 bias | 无 | 有 |
| K 投影 bias | 无 | 有 |
| V 投影 bias | 无 | 有 |
| O 投影 bias | 无 | 无 |

### 3.2 配置结构差异

#### 基础配置 (`kuiper/include/model/config.h`)
```cpp
struct ModelConfig {
  int32_t dim = 0;              // 模型维度
  int32_t hidden_dim = 0;       // FFN 隐藏层维度
  int32_t layer_num = 0;        // 层数
  int32_t head_num = 0;         // 注意力头数
  int32_t kv_head_num = 0;      // KV 头数 (GQA)
  int32_t vocab_size = 0;       // 词表大小
  int32_t seq_len = 0;          // 序列长度
#ifdef QWEN3_SUPPORT
  int32_t immediate_dim_ = 0;   // Qwen3 特有参数
#endif
};
```

#### Qwen3 专用配置 (`kuiper/include/model/qwen3.h`)
```cpp
struct QWen3TransformerConfig {
  int32_t kv_dim_ = 0;
  int32_t kv_mul_ = 0;
  int32_t head_size_ = 0;
  int32_t immediate_size_ = 0;     // Qwen3 特有：中间层维度
  int32_t vocab_size_ = 0;
  int32_t dim_ = 0;
  int32_t hidden_dim_ = 0;
  int32_t layer_num_ = 0;
  int32_t head_num_ = 0;
  int32_t kv_head_num_ = 0;
  int32_t seq_len_ = 0;
  bool is_shared_weight_ = false;  // 是否共享权重
};
```

### 4.3 导出脚本对比

**LLaMA 导出** (`tools/export_llama3.py`):
```python
# Attention weights (无 bias)
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wq.weight)
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wk.weight)
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wv.weight)
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wo.weight)
```

**Qwen2 导出** (`tools/export_qwen2.py`):
```python
# Attention weights (包含 bias)
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wq.weight)
    serialize_fp32(out_file, layer.attention.wq.bias)  # Qwen 有 bias
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wk.weight)
    serialize_fp32(out_file, layer.attention.wk.bias)  # Qwen 有 bias
for layer in model.layers:
    serialize_fp32(out_file, layer.attention.wv.weight)
    serialize_fp32(out_file, layer.attention.wv.bias)  # Qwen 有 bias
```

## 5. 编译与构建

### 5.1 编译选项

```bash
# LLaMA2
cmake -DUSE_CPM=ON ..

# LLaMA3 / Qwen2
cmake -DUSE_CPM=ON -DLLAMA3_SUPPORT=ON ..

# Qwen3（需要额外宏定义）
cmake -DUSE_CPM=ON -DLLAMA3_SUPPORT=ON -DQWEN3_SUPPORT=ON ..
```

### 5.2 模型类继承关系

```
                    ┌─────────────┐
                    │    Model    │ (基类: kuiper/include/model/model.h)
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           │               │               │
    ┌──────┴──────┐ ┌──────┴──────┐ ┌──────┴──────┐
    │ LLama2Model │ │  Qwen2Model │ │  Qwen3Model │
    └─────────────┘ └─────────────┘ └─────────────┘
```

**统一接口：**
- `init()` - 初始化模型
- `predict()` - 预测下一个 token
- `forward()` - 前向传播
- `embedding()` - 词嵌入

## 6. 前向传播流程（相同）

```cpp
for (int32_t layer_idx = 0; layer_idx < config_->layer_num_; ++layer_idx) {
    attention_rms(layer_idx, input);       // Attention 前的 RMSNorm
    attention_qkv(layer_idx, pos_tensor);  // QKV 投影
    attention_mha(layer_idx, pos_tensor);  // 多头注意力
    feed_forward(layer_idx, input);        // 前馈网络
}
cls_logits(input);  // 分类输出
```

## 7. 总结对比表

### 7.1 主要差异

| 特性 | LLaMA 系列 | Qwen 系列 |
|------|-----------|----------|
| Q/K/V Bias | 无 | 有 |
| 配置参数 | 标准 TransformerConfig | Qwen3 有 `immediate_dim_` |
| Tokenizer | LLaMA2: SentencePiece<br>LLaMA3: BPE | BPE |
| 权重文件大小 | 相对较小 | 略大（包含 bias） |
| 共享权重 | 支持 | 支持 (is_shared_weight_) |

### 7.2 共同点

| 特性 | 实现 |
|------|------|
| 核心架构 | Decoder-only Transformer |
| 归一化方式 | RMSNorm |
| 位置编码 | RoPE (Rotary Position Embedding) |
| 激活函数 | SwiGLU |
| 注意力机制 | GQA (Grouped Query Attention) |
| 层结构 | 完全相同 |

## 8. 开发建议

### 8.1 权重转换

从 HuggingFace 导出时：
- LLaMA: 仅导出 weight
- Qwen: 需要同时导出 weight 和 bias

### 8.2 内存规划

对于相同参数量的模型：
- Qwen 模型需要额外内存存储 Q/K/V 的 bias 参数
- 额外内存 ≈ `3 × layer_num × (dim + 2 × kv_dim) × sizeof(float)`

### 8.3 兼容性设计

如需同时支持两种模型：
1. 使用条件编译 (`#ifdef QWEN_SUPPORT`)
2. 或使用运行时配置切换
3. 建议抽象出 `AttentionConfig` 结构体统一管理
