# KuiperLLama - 大模型推理框架（AI Infra）

### 项目概述
KuiperLLama 是一个从零自研的高性能 LLM 推理框架，专注于大语言模型低延迟、高吞吐推理服务。采用模块化架构设计，支持 LLaMA2/3 和 Qwen2.5/3 等主流模型，具备 CUDA 加速和 Int8 量化能力。

---

### 核心技术实现

#### 1. **推理引擎架构设计**
- **分层架构设计**：基础层（Buffer/Tensor）→ 算子层（MatMul/Attention）→ 模型层（LLaMA/Qwen）
- **统一后端抽象**：支持 CPU 和 CUDA 双后端无缝切换，通过 CMake 配置自动选择
- **模块化模型支持**：插件式架构，快速集成新模型（已支持 LLaMA2/3、Qwen2.5/3）

#### 2. **高性能 CUDA 算子实现**
- **核心算子优化**：
  - Matrix Multiply：实现分块矩阵乘法，使用 float4 向量化（128-bit）和共享内存优化
  - Multi-Head Attention：使用 float4 向量化内存访问，优化注意力计算
  - RMSNorm：基于 CUB 库的归约操作，提升计算效率
- **内存优化策略**：
  - 智能内存池：预分配 GPU 内存，减少 malloc 开销
  - 零拷贝优化：主机-设备内存共享机制
  - KV-Cache 优化：实现动态缓存管理，减少重复计算

#### 3. **推理优化策略**
- **批处理调度**：实现动态批处理，根据输入长度自动调整 batch 大小
- **流水线优化**：计算与内存传输重叠执行，提升 GPU 利用率
- **量化支持**：Int8 量化实现，权重量化和激活量化，减少 50%+ 显存占用

#### 4. **工程化实践**
- **构建系统**：CMake + CPM 自动依赖管理，支持第三方库集成
- **测试框架**：完整的单元测试覆盖（Google Test），确保代码质量
- **工具链**：
  - 模型导出工具：HuggingFace 格式转换
  - 推理验证：与 HuggingFace 结果对比验证
  - 性能基准：自动化性能测试

#### 5. **性能指标**
- **推理速度**：RTX 3060 上达到 60+ tokens/s
- **显存优化**：通过 KV-Cache 减少 50%+ 显存占用
- **延迟优化**：P50 < 100ms（batch=1）
- **上下文支持**：支持 4K+ 上下文长度
